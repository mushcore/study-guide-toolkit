1. In what language is Linux written? What does the language have to do with
   the success of Linux?

   Linux is written in C. C made Linux portable -- the same source code can
   be compiled on many different hardware platforms, which is a core reason
   Linux became the dominant server OS.


2. What is a shell? How does it work with the kernel? With the user?

   A shell is a command line interpreter and programming language that runs
   as a user-space program on top of the kernel.

   With the kernel: when a user types a command, the shell makes system calls
   (fork/exec) to create a new process and run the program, then waits for
   the kernel to signal completion.

   With the user: the shell displays a prompt, reads commands from standard
   input, provides variable substitution, filename generation (globbing),
   I/O redirection, pipelines, and a scripting interface.


3. What happens when you give the following commands when the file named done
   already exists (assume to_do also exists)?

   a. cp to_do done
      Copies the contents of to_do into done, overwriting done without
      warning. Both files exist afterward; to_do is unchanged.

   b. mv to_do done
      Moves to_do to done, overwriting done without warning. After the
      command, to_do no longer exists.


4. What happens if you use diff to compare two binary files that are not
   identical? Explain why the diff output for binary files is different than
   the diff output for ASCII files.

   diff outputs: "Binary files file1.gz and file2.gz differ"

   It simply reports that the files differ without showing specifics. diff
   works line-by-line on readable text. Binary files contain non-printable
   characters and no meaningful line structure, so displaying byte-level
   differences would be unreadable.


5. What does which do? What is the result of giving the which utility the name
   of a command that resides in a directory that is not in your search path?

   which searches the directories in PATH and displays the absolute pathname
   of the executable that would be run.

   If the command is not in PATH, which returns an error like:
     /usr/bin/which: no such_command in (...)


6. Experiment by calling the file utility with names of files in /usr/bin.
   How many different types of files can you find there (approximately)?

   Running "file /usr/bin/*" reveals approximately 6-8 types, including:
   ELF 64-bit executables, position-independent executables, POSIX shell
   scripts, Bash scripts, Perl/Python scripts, and symbolic links.

   file determines type by examining file contents (magic numbers), not
   the file extension.


7. Try giving the two commands and explain the difference between them:

   a. echo cat
      Output: cat
      echo prints its arguments as text to stdout.

   b. cat echo
      Output: cat: echo: No such file or directory
      cat tries to open a file named "echo", which doesn't exist.

   The first token on the command line is the command; subsequent tokens
   are its arguments.


8. Find an existing file, or create a file that:

   a. gzip compresses by more than 80 per cent
      A file with highly repetitive content (e.g., a large log file or
      a file of repeated lines) compresses well over 80% because gzip
      excels at eliminating repeated patterns.

   b. gzip compresses by less than 10 per cent
      A compiled binary or already-compressed file (.jpg, .mp3, .gz) has
      very little redundancy, so gzip achieves less than 10%.

   c. gets larger when compressed with gzip
      A very small file (e.g., 3 bytes) grows because gzip adds an 18-byte
      header/trailer, exceeding any compression savings.

12. What command can you use to copy all of the files in the directory lab1
    (including any subdirectories) into the directory /home/eric?

      cp -r lab1 /home/eric/

    The -r option copies recursively, including subdirectories.
