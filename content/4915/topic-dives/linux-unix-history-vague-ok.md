Scope: only the rough order matters for the exam. Exact dates are not required.

-   1969 — UNIX at AT&T Bell Labs (Thompson, Ritchie)
-   1973 — rewritten in C (portability)
-   1983 — GNU started (Stallman)
-   1991 — Linux started (Torvalds)
-   1994 — Linux kernel 1.0
-   1996 — 2.0; 2003 — 2.6

Linux became popular in business for perceived reliability, performance, and security — not because "it's free" alone. (Midterm Q7 answer: E.)

> **Pitfall**
>
> Linux is the *kernel*; GNU is the userland. "Linux" in casual speech usually means "GNU/Linux distribution", which is the kernel + GNU tools + package manager. The distinction matters when debugging — a kernel bug is not the same as a coreutils bug.

> **Takeaway**: UNIX (Bell Labs, 1970s) → BSD + System V → Linux kernel (Torvalds, 1991) + GNU userland (Stallman) = modern Linux. Relative order matters; exact dates do not. Torvalds + Stallman are the two names the exam wants.
