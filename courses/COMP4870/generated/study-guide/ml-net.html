2 · Core concepts

Remember / understand level. One definition, one analogy, one snippet. Source tags are traceable to the slides or scripts.

MLContext

Entry point for everything. Holds trainer catalogs, transforms, random seed. Analogy: the DbContext of ML.NET.
mlContext = new MLContext(seed: 0);
Supplying seed: 0 makes the environment deterministic — same training data → same model every run.
source: VS Code script line 42-44

IDataView

Tabular, schematized, potentially-lazy in-memory data. Analogy: a LINQ-style wrapper around a spreadsheet.
IDataView dataView = mlContext.Data
    .LoadFromTextFile<TaxiTrip>(dataPath, hasHeader: true, separatorChar: ',');
source: VS Code script line 50

LoadFromTextFile<T>

Loads a CSV (or TSV) into an IDataView. Generic T is your schema class with [LoadColumn(n)] attributes.
mlContext.Data.LoadFromTextFile<TaxiTrip>(
    _trainDataPath, hasHeader: true, separatorChar: ',');
source: VS Code script line 50

Schema class + [LoadColumn]

A POCO decorated with [LoadColumn(index)] tells ML.NET how to map CSV columns to fields. Index is zero-based.
public class TaxiTrip {
    [LoadColumn(0)] public string VendorId;
    [LoadColumn(5)] public string PaymentType;
    [LoadColumn(6)] public float FareAmount; // label
}
source: slide 24 + VS Code script

Features vs Label

Features are what we learn from (inputs). Label is what we predict (the answer). In taxi-fare: fare_amount is Label; everything else except trip_time_in_secs is a feature.
Exclude trip_time_in_secs: you don't know trip time before the trip, so you can't use it to predict fare in production.
source: VS Code + VS2022 script tables

CopyColumns → "Label"

Many trainers expect the column literally named Label. Copy your real label column into a column called "Label".
mlContext.Transforms.CopyColumns(
    outputColumnName: "Label", inputColumnName: "FareAmount")
source: VS Code script line 52; slide 26

One-Hot Encoding

Turns a categorical string (e.g. "CSH", "CRD") into a numeric vector. Trainers need numbers, not strings.
.Append(mlContext.Transforms.Categorical
    .OneHotEncoding(outputColumnName: "PaymentTypeEncoded",
                    inputColumnName: "PaymentType"))
source: VS Code script line 54-56; slide 27

Concatenate → "Features"

Combines all feature columns into a single vector column named Features (what trainers expect by default).
.Append(mlContext.Transforms.Concatenate(
    "Features",
    "VendorIdEncoded", "RateCodeEncoded",
    "PassengerCount", "TripDistance",
    "PaymentTypeEncoded"))
source: VS Code script line 58; slide 28

Trainer: FastTree

Gradient-boosted decision-tree regressor. The go-to starting regression trainer in the slides.

Needs the NuGet Microsoft.ML.FastTree — extra package beyond Microsoft.ML.

Tweedie-loss variant — best for skewed non-negative targets (fare amounts, insurance claims). Model Builder picked this as the best trainer at 0.93 accuracy.

pipeline.Fit(dataView)

Kicks off training. Returns an ITransformer — the trained model.
var model = pipeline.Fit(dataView);
Fit = learns from data. Transform = applies the trained transform to new data.

Model.Save / Model.Load

Persist the trained model + schema to a .zip; load it back later for inference without retraining.
mlContext.Model.Save(model, dataView.Schema, _modelPath);
// …
ITransformer loaded = mlContext.Model.Load(_modelPath, out DataViewSchema schema);

CreatePredictionEngine

Makes a single-row prediction helper. Strongly typed on input and output classes.
var engine = mlContext.Model
    .CreatePredictionEngine<TaxiTrip, TaxiTripFarePrediction>(model);
var prediction = engine.Predict(taxiTripSample);
For batch inference use model.Transform(dataView) instead.

Evaluate — regression metrics

RSquared: 0→1, higher is better. RootMeanSquaredError: lower is better. Use test data the model has never seen.
var predictions = model.Transform(testDataView);
var metrics = mlContext.Regression.Evaluate(predictions, "Label", "Score");
Console.WriteLine(metrics.RSquared);
Console.WriteLine(metrics.RootMeanSquaredError);

Train() — the full pipeline (MEMORIZE THIS)

This is the exam-likely 10-line skeleton. Five transforms + one trainer + Fit.
ITransformer Train(MLContext mlContext, string dataPath) {
    IDataView dataView = mlContext.Data
        .LoadFromTextFile<TaxiTrip>(dataPath, hasHeader: true, separatorChar: ',');

    var pipeline = mlContext.Transforms.CopyColumns(
            outputColumnName: "Label", inputColumnName: "FareAmount")
        .Append(mlContext.Transforms.Categorical.OneHotEncoding(
            "VendorIdEncoded", "VendorId"))
        .Append(mlContext.Transforms.Categorical.OneHotEncoding(
            "RateCodeEncoded", "RateCode"))
        .Append(mlContext.Transforms.Categorical.OneHotEncoding(
            "PaymentTypeEncoded", "PaymentType"))
        .Append(mlContext.Transforms.Concatenate("Features",
            "VendorIdEncoded", "RateCodeEncoded",
            "PassengerCount", "TripDistance", "PaymentTypeEncoded"))
        .Append(mlContext.Regression.Trainers.FastTree());

    var model = pipeline.Fit(dataView);
    mlContext.Model.Save(model, dataView.Schema, _modelPath);
    return model;
}

Evaluate()

Transform the test data, call Regression.Evaluate, print RSquared and RootMeanSquaredError.
void Evaluate(MLContext mlContext, ITransformer model) {
    IDataView dataView = mlContext.Data
        .LoadFromTextFile<TaxiTrip>(_testDataPath, hasHeader: true, separatorChar: ',');
    var predictions = model.Transform(dataView);
    var metrics = mlContext.Regression.Evaluate(predictions, "Label", "Score");

    Console.WriteLine($"RSquared:  {metrics.RSquared:0.##}");
    Console.WriteLine($"RMSE:      {metrics.RootMeanSquaredError:#.##}");
}
Example output from the script: RSquared: 0.89, RMSE: 3.3.

TestSinglePrediction()

Use CreatePredictionEngine<TIn, TOut> to wrap the model for a single input.
void TestSinglePrediction(MLContext mlContext, ITransformer model) {
    var engine = mlContext.Model
        .CreatePredictionEngine<TaxiTrip, TaxiTripFarePrediction>(model);

    var sample = new TaxiTrip {
        VendorId = "VTS", RateCode = "1",
        PassengerCount = 1, TripTime = 1140,
        TripDistance = 3.75f, PaymentType = "CRD",
        FareAmount = 0   // to predict. actual = 15.5
    };
    var prediction = engine.Predict(sample);
    Console.WriteLine($"Predicted fare: {prediction.FareAmount:0.####}, actual: 15.5");
}

Consume the saved model

Production path: skip training, just Model.Load and predict.
void LoadAndUseModel() {
    mlContext = new MLContext(seed: 0);
    DataViewSchema modelSchema;
    ITransformer trainedModel = mlContext.Model.Load(_modelPath, out modelSchema);
    TestSinglePrediction(mlContext, trainedModel);
}

4.1 · The 10-line regression skeleton (taxi fare)

var mlContext = new MLContext(seed: 0);
IDataView data = mlContext.Data
    .LoadFromTextFile<TaxiTrip>(path, hasHeader: true, separatorChar: ',');

var pipeline = mlContext.Transforms.CopyColumns("Label", "FareAmount")

var engine = mlContext.Model

    PaymentType = "CRD", FareAmount = 0

var pred = engine.Predict(sample);
Console.WriteLine(pred.FareAmount);

8 · Mnemonics & memory aids

6 scenarios → "CIRROF-D"

C I R R O F + D
Classification · Image classification · Regression (value prediction) · Recommendation · Object detection · Forecasting · ("D" = data text classification = a flavor of Classification). Also memorable as "Cirrus clouds rain over forecasts".

Pipeline order → "CONCAT-F"

C · O · N · C · A · T · F
CopyColumns (Label) → OneHotEncoding → N(more encoding) → Concatenate (Features) → Append trainer → Train via Fit → Finalize via Save. When in doubt, CONCAT before the trainer.

Regression metrics → "R² & RMSE"

R² ↑ good · RMSE ↓ good
RSquared: 0→1, closer to 1 = better (variance explained). RootMeanSquaredError: same units as label, smaller = better. Both come off mlContext.Regression.Evaluate.

Fit vs Transform → "Learn vs Apply"

Fit = Learn · Transform = Apply
Fit runs once on training data — produces an ITransformer. Transform runs many times on new data — consumes the ITransformer. Prediction = Transform under the hood.

Two NuGets → "Core + Tree"

Microsoft.ML + Microsoft.ML.FastTree
Every taxi-fare demo needs both. Forget FastTree → you get a missing-type compile error on mlContext.Regression.Trainers.FastTree().

CLI scenarios → "CRIFR"

C · R · I · F · R
AutoML CLI commands: classification · regression · image-classification · forecasting · recommendation. "Crispy Fried Rice" if it helps.

9 · Gotchas & exam traps

[LoadColumn] is zero-indexed

[LoadColumn(0)] is the first CSV column. In AutoML CLI, --label-col 6 is the seventh column. Same convention.

Features column must be named "Features"

After Concatenate, the output column must be "Features" (the trainer's default input name). Rename it and you need to pass featureColumnName: to the trainer.

Label column gets renamed too

Your real label (FareAmount) must be copied into a column literally called "Label" via CopyColumns. Otherwise the trainer defaults don't find it.

FastTree vs FastTreeTweedie

Both are regression trainers. FastTreeTweedie handles skewed, non-negative targets (fare amounts, insurance claims) better. Model Builder often picks Tweedie as the winner for taxi fare.

One-Hot Encoding is mandatory for strings

Trainers accept only numbers / vectors. If a feature is a string (VendorId, PaymentType), you must OneHotEncoding it before Concatenate, or the pipeline fails to build.

Don't include leaky features

trip_time_in_secs is excluded from features because you don't know it before the trip. Using it would be data leakage — the model looks great in training but fails in production.

MLContext(seed: 0) ≠ fully deterministic if you skip it

Without a seed, you get different results each run. new MLContext(seed: 0) locks the RNG so results are reproducible — essential for demos and exams.

Model.Save needs the schema

The signature is Save(model, dataView.Schema, path) — three args. Drop the schema and you can't Load it back without the original class.

Prediction output uses [ColumnName("Score")]

For regression, the predicted numeric column is always called Score. Tag your output POCO's field with [ColumnName("Score")] to map it.

CreatePredictionEngine is single-row

For one prediction per call → CreatePredictionEngine. For a batch of many rows → model.Transform(dataView). Don't loop a prediction engine when you could batch.

CopyToOutputDirectory = PreserveNewest

If your CSV isn't showing up in bin/Debug/net6.0/Data, you forgot the <None Update … CopyToOutputDirectory> block in .csproj.

mlnet CLI is NOT the same as Microsoft.ML

mlnet is a dotnet global tool for AutoML at the command line. Microsoft.ML is the NuGet package you reference from code. Different artifacts, same underlying library.
