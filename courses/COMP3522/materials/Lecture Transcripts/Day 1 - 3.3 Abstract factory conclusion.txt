 You're going to want to use the abstract factory pattern if you want to work with families of related objects. And also, if the exact number of product families are unknown and you want to add new ones in the future. And also, just like the factory design pattern, the abstract factory pattern also follows many of the same solid design principles. As for some of the disadvantages, just like before, there are a lot of classes and interfaces you must work with. And because of this, they can make the code more complex and harder to debug. And also, just like the factory pattern, a lot of the classes can get kind of artificial, where you have to create whole new subclasses every time you want to make a minor change to the object creation process.
