Comp 4915 - Lab 2: Unix File System
====================================

Procedures:
  mkdir ~/Assignments
  touch ~/Assignments/lab2.txt
  chmod 600 ~/Assignments/lab2.txt


1) What commands did you use to create the file, and to change the
   permissions on your file?

     mkdir ~/Assignments
     touch ~/Assignments/lab2.txt
     chmod 600 ~/Assignments/lab2.txt

   chmod 600 sets permissions to rw------- (read/write for owner only).


2) How are directories different from ordinary files? How can they be
   distinguished using the ls utility?

   A directory is a special file containing inode/filename pairs. An
   ordinary file contains user data.

   Using ls -l, the first character distinguishes them:
     d = directory (drwxr-xr-x)
     - = ordinary file (-rw-r--r--)


3) Is each of the following an absolute pathname, a relative pathname, or a
   simple filename:

   a) milk_co                          Simple filename (no slashes).
   b) corresp/business/milk_co         Relative pathname (has slashes, no leading /).
   c) /home/alex                       Absolute pathname (starts with /).
   d) /home/alex/literature/promo      Absolute pathname (starts with /).
   e) ..                               Relative pathname (parent directory).


4) List the commands you can use to:

   a) Make your home directory the working directory:
        cd

   b) Identify the working directory:
        pwd


5) If your working directory is /home/alex/assn4, what are two different
   commands that you can use to create a subdirectory named classics?

     mkdir classics
     mkdir /home/alex/assn4/classics

   To remove classics and its contents:
     rm -r classics


6) The ls -i command displays a filename preceded by the inode number of the
   file. Write a command to output inode/filename pairs for the files in the
   working directory, sorted by inode number.

     ls -i | sort -n


7) What specific command would you use to mount the /boot and / directories
   which are on /dev/sda?

     mount /dev/sda2 /mnt/root
     mount /dev/sda1 /mnt/boot

   The mount command requires root privileges. Mount point directories
   must exist before mounting. Actual partition numbers depend on the
   disk layout (check with fdisk -l).


8) Use the df utility to answer the following questions:

   a) How many file systems are there on your Linux system?

      Run: df
      A typical system has approximately 5-10 mounted file systems
      (root, /boot, /home, plus virtual filesystems like tmpfs, devtmpfs).

   b) What file system stores your home directory?

      Run: df ~

   c) Attempt to create a hard link to a file on another file system.
      What error message do you get? What happens with a symbolic link?

      Hard link:
        $ ln /boot/vmlinuz ~/vmlinuz-link
        ln: failed to create hard link: Invalid cross-device link

      Hard links cannot cross filesystem boundaries because they reference
      inodes, which are unique per filesystem.

      Symbolic link:
        $ ln -s /boot/vmlinuz ~/vmlinuz-link
        (succeeds)

      Symbolic links can cross filesystems because they store the target
      pathname as text, not an inode reference.


9) Examining /etc/passwd:

   a) Field separator: colon ( : )

   b) Fields per user: 7
      (login name, password/x, UID, GID, GECOS, home directory, login shell)

   c) How many users:
      Run: wc -l /etc/passwd

   d) How many login shells in use:
      Run: cut -d: -f7 /etc/passwd | sort -u

   e) Does your system use shadow passwords?
      If the second field in /etc/passwd contains "x", then yes -- actual
      hashes are stored in /etc/shadow instead.

   f) Protections on /etc/passwd:
      -rw-r--r-- 1 root root (644). World-readable because programs need
      to look up user information.

   g) Protections on /etc/shadow:
      ---------- 1 root root (000 or 400). Only root can read it because
      it contains hashed passwords.


10) If /home/jenny/draft and /home/alex/letter are links to the same file
    and the following sequence occurs, what will the date be?

    The date will be February 1, 2004. Both editors loaded the file into
    separate memory buffers. Jenny wrote first (Jan 31), then Alex
    overwrote with his buffer (Feb 1). Alex writes last, so his date
    remains.

    e) To stop sharing changes: copy the file and remove the link:
       cp letter letter.tmp
       rm letter
       mv letter.tmp letter


11) Are there any subdirectories of the root directory that you cannot search
    or read from a normal user account?

    Yes. /root typically has permissions drwx------ (700), so normal users
    cannot search (cd into) or read (ls) its contents. /lost+found may
    also be similarly restricted.


12) Suppose that a user belongs to a group that has all permissions on a file
    named jobs_list, but the user, as the owner, has no permissions.

    The user cannot read, write, or execute the file. Linux checks owner
    permissions first and stops -- group permissions are never checked for
    the owner.

    However, the owner can always change permissions:
      chmod u+rwx jobs_list


13) Create a file named -x in an empty directory. Explain what happens when
    you try to rename it. How can you rename it?

    Create:  touch -- -x
    Rename fails:  mv -x newname  (mv interprets -x as an option)
    Fix:  mv -- -x newname

    The -- tells mv that no more options follow, so -x is treated as a
    filename.


14) Suppose that the working directory contains a single file named andor.

    a) $ mv andor and\/or
       Error: mv: cannot move 'andor' to 'and/or': No such file or directory
       The shell interprets and/or as a path -- file "or" inside directory
       "and", which doesn't exist.

    b) If a directory named "and" exists, the command succeeds and moves
       andor into and/or.


15) Is it possible to distinguish a file from a hard link to the file?

    No. Both are equal directory entries pointing to the same inode. There
    is no "original" -- they are indistinguishable. The only clue is a link
    count greater than 1 in ls -l.


16) Create 2 files:

    $ man bash > wiener; man bash | col -b > sausage

    b) wiener is larger than sausage. Man pages use backspace characters
       for bold/underline formatting. wiener contains these backspace
       sequences; sausage has them stripped by col -b, resulting in a
       smaller, cleaner file.


17) What does col -b do?

    col -b strips backspace characters and overprinted characters from
    input, producing clean plain text from formatted man page output.


18) Delete wiener, rename sausage to dog, create a symbolic link:

      rm wiener
      mv sausage dog
      ln -s dog wiener
