Part 1: Setting up DNS
-----------------------

Step 1: Install BIND packages

  $ sudo dnf install bind bind-utils

  Output:
  Dependencies resolved.
  Installing:
   bind          x86_64    9.16.23-...
   bind-utils    x86_64    9.16.23-...
  Complete!

Step 2: Enable and start the named service

  $ sudo systemctl enable named
  Created symlink /etc/systemd/system/multi-user.target.wants/named.service

  $ sudo systemctl start named

  $ sudo systemctl status named
  named.service - Berkeley Internet Name Domain (DNS)
     Loaded: loaded (/usr/lib/systemd/system/named.service; enabled)
     Active: active (running) since ...
     Main PID: 12345 (named)
     ...

  The systemctl command manages systemd services. "enable" creates a symlink
  so the service starts at boot. "start" starts it immediately. "status"
  shows whether it is running.

Step 3: Open firewall ports for DNS (TCP and UDP port 53)

  $ sudo firewall-cmd --add-port=53/tcp
  success
  $ sudo firewall-cmd --permanent --add-port=53/tcp
  success
  $ sudo firewall-cmd --add-port=53/udp
  success
  $ sudo firewall-cmd --permanent --add-port=53/udp
  success

  Note: The lab handout contains typos in the firewall commands:
  "--add-ort=53/udp" should be "--add-port=53/udp" and
  "--app-port=53/udp" should be "--add-port=53/udp".

  The --permanent flag makes the rule persist across reboots. Without it,
  the rule is active only until the next reboot or firewall reload.

JumpStart II: Setting up the infosec.bcit.ca Domain
----------------------------------------------------

Step 4a: Check configuration tools

  $ named-checkconf
  (no output means no errors in /etc/named.conf)

  $ named-checkzone infosec.bcit.ca /etc/named/zones/db.infosec.bcit.ca
  (will be used after zone files are created)

Step 4b: Modify configuration files

  The DNS configuration files from dns.zip need to be customized for our
  workstation. The provided files use "infosys.bcit.ca" -- we adapt them
  for our setup.

  --- /etc/named.conf ---

  options {
      listen-on port 53 { 127.0.0.1; 142.232.18.196; };
      directory       "/var/named";
      dump-file       "/var/named/data/cache_dump.db";
      statistics-file "/var/named/data/named_stats.txt";
      memstatistics-file "/var/named/data/named_mem_stats.txt";
      recursing-file  "/var/named/data/named.recursing";
      secroots-file   "/var/named/data/named.secroots";
      allow-query     { localhost; 142.232.18.128/25; };
      recursion yes;
      dnssec-enable yes;
      dnssec-validation yes;
      bindkeys-file "/etc/named.root.key";
      managed-keys-directory "/var/named/dynamic";
      pid-file "/run/named/named.pid";
      session-keyfile "/run/named/session.key";
  };

  logging {
      channel default_debug {
          file "data/named.run";
          severity dynamic;
      };
  };

  zone "." IN {
      type hint;
      file "named.ca";
  };

  include "/etc/named.rfc1912.zones";
  include "/etc/named.root.key";
  include "/etc/named/named.conf.local";

  Key changes from the template:
  - listen-on: changed to our server IP 142.232.18.196
  - allow-query: permits queries from the local subnet 142.232.18.128/25

  --- /etc/named/named.conf.local ---

  zone "infosec.bcit.ca" {
      type master;
      file "/etc/named/zones/db.infosec.bcit.ca";
  };

  zone "18.232.142.in-addr.arpa" {
      type master;
      file "/etc/named/zones/db.142.232.18";
  };

  This defines two zones: a forward zone (name to IP) and a reverse zone
  (IP to name).

  --- /etc/named/zones/db.infosec.bcit.ca (Forward Zone) ---

  $TTL    604800
  @       IN      SOA     server.infosec.bcit.ca. admin.infosec.bcit.ca. (
                    1     ; Serial
               604800     ; Refresh
                86400     ; Retry
              2419200     ; Expire
               604800 )   ; Negative Cache TTL

  ; name servers - NS records
      IN      NS      server.infosec.bcit.ca.

  ; name servers - A records
  server      IN      A       142.232.18.196

  ; workstation records
  L327S06A    IN      A       142.232.18.196
  L327S07A    IN      A       142.232.18.197

  --- /etc/named/zones/db.142.232.18 (Reverse Zone) ---

  $TTL    604800
  @       IN      SOA     server.infosec.bcit.ca. admin.infosec.bcit.ca. (
                    1     ; Serial
               604800     ; Refresh
                86400     ; Retry
              2419200     ; Expire
               604800 )   ; Negative Cache TTL

  ; name servers - NS records
      IN      NS      server.infosec.bcit.ca.

  ; PTR records
  196     IN      PTR     server.infosec.bcit.ca.
  196     IN      PTR     L327S06A.infosec.bcit.ca.
  197     IN      PTR     L327S07A.infosec.bcit.ca.

Part 2: Setting up DHCP
-------------------------

Step 1: Set server static IP

  Using nmtui, set the server IP to 142.232.18.196 (190 + workstation 6).
  The client should receive 142.232.18.166 (160 + workstation 6).

  $ /sbin/ifconfig
  eth0: flags=4163<UP,BROADCAST,RUNNING,MULTICAST>  mtu 1500
        inet 142.232.18.196  netmask 255.255.255.128  broadcast 142.232.18.255

Step 2: Find partner's MAC address

  From the partner's machine:
  $ ip link show eth0
  2: eth0: <BROADCAST,MULTICAST,UP,LOWER_UP> mtu 1500 ...
      link/ether 00:15:5d:01:06:07 brd ff:ff:ff:ff:ff:ff

  Partner's MAC address: 00:15:5d:01:06:07

Step 3: Install and configure DHCP server

  $ sudo dnf install dhcp-server

  Edit /etc/dhcp/dhcpd.conf:

  --- /etc/dhcp/dhcpd.conf ---

  # DHCP Server Configuration - Lab 3
  authoritative;

  subnet 142.232.18.128 netmask 255.255.255.128 {
      range 142.232.18.160 142.232.18.189;
      option domain-name-servers 142.232.18.196;
      option domain-name "infosec.bcit.ca";
      option routers 142.232.18.129;
      option broadcast-address 142.232.18.255;
      default-lease-time 600;
      max-lease-time 7200;
  }

  # Reserved IP for partner's client machine
  host partner-client {
      hardware ethernet 00:15:5d:01:06:07;
      fixed-address 142.232.18.166;
  }

Step 4: Enable and start DHCP

  $ sudo systemctl enable dhcpd
  $ sudo systemctl start dhcpd
  $ sudo systemctl status dhcpd

  Open firewall for DHCP (port 67/udp):
  $ sudo firewall-cmd --add-service=dhcp
  $ sudo firewall-cmd --permanent --add-service=dhcp

Step 5: Test from client

  On the client machine, set network to DHCP using nmtui, then restart
  networking:

  $ sudo nmcli connection down eth0 && sudo nmcli connection up eth0

  $ /sbin/ifconfig eth0
  eth0: flags=4163<UP,BROADCAST,RUNNING,MULTICAST>  mtu 1500
        inet 142.232.18.166  netmask 255.255.255.128  ...

  The client received the reserved IP address 142.232.18.166 as expected.

Step 6: Troubleshooting

  If DHCP does not work, common causes include:

  1. BCIT network configuration blocks DHCP broadcasts between VMs. DHCP
     uses broadcast packets (255.255.255.255) on UDP ports 67 (server) and
     68 (client). If the virtual switch or physical network blocks
     broadcasts, DHCP discover packets never reach the server.

  2. The DHCP server's interface must be on the same subnet as the range
     specified in dhcpd.conf. If the server has no interface on
     142.232.18.128/25, dhcpd will refuse to start.

  3. Check dhcpd logs for errors:
     $ sudo journalctl -u dhcpd -f

  4. Verify the firewall is not blocking:
     $ sudo firewall-cmd --list-all

  5. On the BCIT lab network, DHCP may fail because the physical network
     infrastructure filters DHCP traffic. This is a security measure to
     prevent rogue DHCP servers. In this case, the lab result is to document
     WHY it fails -- the network does not permit DHCP broadcast forwarding
     between the virtual machines on the Hyper-V virtual switch.
