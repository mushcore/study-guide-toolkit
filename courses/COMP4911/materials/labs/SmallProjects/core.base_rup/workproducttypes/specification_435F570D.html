Work Product Kind: Specification
These work products define the interfaces between different parts of the project, especially between different domains. They define exactly what something is or what it must do. For example, the Software Requirements Specification defines how the business goals are realized with system requirements; the Software Architecture Document shows how system requirements are mapped into key aspects of the design.
