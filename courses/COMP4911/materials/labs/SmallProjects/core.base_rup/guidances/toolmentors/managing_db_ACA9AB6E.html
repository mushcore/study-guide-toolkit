Compare and Synchronize

Rational XDE compares a Data Model with an existing DDL script file or database schema. (See Data Model Comparison .) The differences can then be synchronized between the model and the DDL or database. (See Data Model Synchronization .) You initiate this comparison and synchronization process using the Data Model to Database Synchronization Wizard. (See Comparing and Synchronizing the Data Model .)

Rational XDE requires you to compare and synchronize your Data Model from the database level. The DDL or database schema that you are comparing must use the same name as your Data Model database. You must also compare to a DDL or database that uses the same database management system (DBMS) as the Data Model database.

Refer to the Rational XDE online Help topic Data Model Compare and Synchronization .

A connection with the database is needed for the compare and synchronization process. The following Rational XDE online Help topics provide information on establishing database connections for a specific DBMS:

  • Oracle Database Connections
  • SQL Server Database Connections
  • Sybase Database Connections
  • DB2 Database Connections
