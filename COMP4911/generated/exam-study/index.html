Instructor intel — Bruce Link

  • Review slides ≈ 80% of final. The two review decks are the primary anchor.
  • Quiz questions reappear from the same bank. Study quiz 3 + quiz 4 verbatim.
  • Easier to BS, partial credit achievable. Write SOMETHING for every short answer / essay.

Priorities

What Bruce flagged, what to skip, and what the quizzes revealed.

HIGH priority — study first

3 session bean types (SLSB / SFSB / Singleton)
Bruce will ask. Lifecycle differences + @Stateless / @Stateful / @Singleton annotations. Ch 5-7.
Container-managed concurrency locks (Singletons)
@Lock(READ) / @Lock(WRITE), @AccessTimeout, @ConcurrencyManagement. Ch 7.
7 entity relationship types
"Technically difficult — can ask hard questions." Owning vs inverse, cascade, fetch. Ch 11.
Composite keys (2 ways)
@EmbeddedId / @Embeddable vs @IdClass — understand which by name, not memorize syntax blindly.
JPQL / queries
"Students struggle here; fair game on final." Practice SELECT, JOIN, FETCH JOIN, named params. Ch 13.
JMS messaging models
Point-to-point (Queue, 1:1) vs publish-subscribe (Topic, 1:many). Review this.
Transaction attributes (6 types)
Quiz 4 hit hard: Required, RequiresNew, Supports, NotSupported, Mandatory, Never. Know caller-has-tx vs no-tx behavior.
Code annotation discipline
@Entity first, then members. Don't forget @Id. Check relationship direction. Graded on annotation completeness, not style.
ACID properties
Quiz 4 asked all four separately: Atomic, Consistent, Isolated, Durable.
MBTI preferences
Quiz 4 had ~8 MBTI questions — high probability on final. E/I, S/N, T/F, J/P distinctions.

LOW priority — skip if tight

4+1 architecture view — NOT on final
Appeared on midterm; Bruce confirmed won't reappear. Know lightly.
Message-Driven Beans deep details (Ch 8)
Low question count. Know: @MessageDriven, onMessage(), Topic vs Queue. Skip lifecycle minutiae.
Specific auto-generated key types
"Several kinds exist" is enough. Won't ask which specific strategy does what.
Java EE Design (Ch 18)
Only if emphasized in lecture. Session Facade / DTO patterns — one-liners at most.
Disaster Recovery, Classic Mistakes
Supplementary. Know categories (People / Process / Product / Technology). Don't memorize lists.
